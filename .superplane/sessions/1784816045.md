# Plan — Issue #6285: Improve AGENTS.md

## Problem
The root `AGENTS.md` is content-rich but hard to scan and is missing several
items the issue asks for. Gaps vs. acceptance criteria:

- No project overview (a reader can't tell what SuperPlane is).
- No prerequisites / language versions (Go, Node, Docker).
- Setup steps are duplicated: the "Build, Test, and Development Commands"
  section and the "Cursor Cloud specific instructions" section both explain
  `dev.up` / `dev.setup` / `dev.server`.
- No "files/directories agents must not modify" section (generated code,
  secrets, vendored deps).
- Ordering buries setup below PR rules; headings could be more scannable.

## Goal
Restructure `AGENTS.md` so it reads top-to-bottom as: what → set up → run →
conventions → what not to touch → how to submit. Keep every still-accurate
rule already present (DB transaction guidelines, models layout, proto rules,
coding style). No content of value is dropped.

## Approach
Rewrite `AGENTS.md` with this section order:

1. Project Overview (2–3 lines, sourced from README).
2. Repository Layout (existing structure list, tightened).
3. Prerequisites & Setup — Go 1.26.2, Node/npm, Docker; the Docker-based
   `dev.up` → `dev.setup` → `dev.server` flow stated once.
4. Build, Test & Lint — exact commands (unchanged).
5. Coding Style & Naming (unchanged).
6. Database Transaction Guidelines + Model layout/API shape (unchanged).
7. Do Not Modify — generated code (`pkg/protos`, `pkg/openapi_client`,
   `web_src/src/api-client`, `api/` swagger), migrations (generate via make),
   secrets/`.env`, vendored deps (`node_modules`, `tmp/go`). New section.
8. Commit & Pull Request Guidelines (Conventional Commits + DCO, required
   checks before submitting) — moved to the end.
9. Cursor Cloud environment notes — reduced to only environment-specific
   gotchas (manual `dockerd`, socket perms, owner setup), no longer repeating
   the generic setup commands.

## Generated-code facts (verified)
- `pkg/protos/` — Go generated from `protos/*.proto`.
- `pkg/openapi_client/` — generated Go CLI SDK (`make openapi.client.gen`).
- `web_src/src/api-client/` — generated TS web SDK (`make openapi.web.client.gen`).
- `api/` — OpenAPI/swagger spec output.
- All regenerated via `make pb.gen`; never hand-edit.

## Tradeoffs
- Pros: matches all acceptance criteria; single source of truth for setup;
  clear "do not touch" boundary reduces agent mistakes on generated code.
- Cons: full rewrite risks reordering churn in git blame; mitigated by
  preserving exact wording of the detailed rule blocks.
- Non-goal: no changes to `web_src/AGENTS.md` (already scoped and current);
  root file links to it.

## Deliverable
- Updated root `AGENTS.md`.
- Committed to branch with a `docs:` message and DCO sign-off.
