# Issue 6276 — Submit commit form on enter

## Problem
In the "Commit changes" dialog (CommitStagingDialog), the user must click the
Commit button to submit. The reporter expects to submit by pressing Enter or
Cmd/Ctrl+Enter while typing the commit message, matching muscle memory from
other apps.

## Location
web_src/src/pages/app/CommitStagingDialog.tsx — the commit message `<Textarea>`.

## Approach
Add an `onKeyDown` handler on the textarea:
- Enter (and Cmd/Ctrl+Enter) submits the form when a non-empty message is
  present and the dialog is not pending.
- Shift+Enter inserts a newline (preserves multi-line commit messages).

Extract a small `submit()` helper and a `canSubmit` guard reused by both the
Commit button and the keyboard handler so behavior stays consistent.

This mirrors the existing keyboard-submit pattern already used in
console/MarkdownPanelEditor.tsx and HtmlPanelEditor.tsx.

## Pros / Cons
- Pro: Meets user expectation for both Enter and Cmd+Enter; consistent with
  existing editors in the codebase.
- Pro: Shift+Enter still allows multi-line messages.
- Con: Plain Enter submitting a textarea is slightly unconventional, but it is
  explicitly what the reporter asked for and the message is typically one line.

## Tests
Add CommitStagingDialog.spec.tsx covering: Enter submits, Cmd+Enter submits,
Shift+Enter does not submit, and empty/pending states do not submit.
