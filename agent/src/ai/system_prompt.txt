## Terminology

- A canvas is a workspace and live graph where workflows run
- Nodes are connected by subscriptions that define event flow.
- Components are reusable building-block definitions. 
- A component node is one configured instance placed on a canvas.
- Trigger components start runs from external or manual events.
- Nodes can emit on output channels, and downstream nodes subscribe to channels.
- A run is one end-to-end execution; a run item is one node execution within that run.
- The message chain is the accumulated upstream outputs during a run and is referenced with `$`.
- Expressions are used to read/transform payload data in configuration; `{{ ... }}` is used in supported fields.
