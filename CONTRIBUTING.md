# Contributing to SuperPlane

Thank you for your interest in contributing to SuperPlane! We welcome contributions from the community.

## Ways to Contribute

There are many ways to contribute to SuperPlane:

- **[Report bugs](docs/contributing/how-to-report-bugs.md)** - Help us identify and fix issues
- **[Suggest features](#creating-issues)** - Share your ideas for new functionality
- **[Write code](docs/contributing/development-workflow.md)** - Fix bugs, implement features, or improve existing code
- **[Improve documentation](docs/contributing/guidelines.md)** - Help make our docs clearer and more comprehensive
- **[Share feedback](#questions)** - Let us know what works well and what could be better

## Development Quickstart

1. **Check existing issues** - Search for similar issues before creating a new one
2. **Create an issue** (if needed) - Report bugs, request features, or ask questions
3. **Fork and clone** - Fork the repository and clone your fork locally
4. **Set up development environment** - Follow the [Development Workflow](docs/contributing/development-workflow.md)
5. **Make your changes** - Create a branch and implement your changes
6. **Submit a pull request** - Follow our [Pull Request guidelines](docs/contributing/pull-requests.md)

## Creating Issues

Before creating an issue, please:

- Search existing issues to see if it's already been reported
- Check closed issues as it may have been resolved
- Review the [documentation](docs/README.md) for answers

When creating an issue, include:

- Clear description of the problem or feature request
- Steps to reproduce (for bugs)
- Relevant context about your environment or use case
- Screenshots if applicable
