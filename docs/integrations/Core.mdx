## Approval

### Configuration

<a id="http-request"></a>

## HTTP Request

Make HTTP requests

### Configuration

| Name | Label | Type | Required | Description |
| --- | --- | --- | --- | --- |
| method | Method | select | yes | - |
| url | URL | string | yes | - |
| queryParams | Query Params | list | no | Query parameters to append to the URL |
| headers | Headers | list | no | Custom headers to send with this request |
| contentType | Body | select | no | Body content type for POST, PUT, and PATCH requests |
| json | JSON Payload | object | no | The JSON object to send as the request body |
| formData | Form Data | list | no | Key-value pairs to send as form data |
| text | Text Payload | text | no | Plain text to send as the request body |
| xml | XML Payload | xml | no | XML content to send as the request body |
| successCodes | Overwrite success definition | string | no | Comma-separated list of success status codes (e.g., 200, 201, 2xx). Leave empty for default 2xx behavior |
| timeoutStrategy | Set Timeout and Retries | select | no | Configure timeout and retry behavior for failed requests |
| timeoutSeconds | Timeout (seconds) | number | no | Timeout in seconds for each request attempt |
| retries | Retries | number | no | Number of retry attempts. Wait longer after each failed attempt (timeout capped to 120s) |

### Example Output

```json
{
  "data": {
    "body": {
      "message": "ok"
    },
    "error": "Error to read request body: EOF",
    "headers": {
      "Content-Type": [
        "application/json"
      ]
    },
    "status": 200
  },
  "timestamp": "2026-01-16T17:56:16.680755501Z",
  "type": "http.request.finished"
}
```

<a id="wait"></a>

## Wait

Wait for a certain amount of time

### Configuration

| Name | Label | Type | Required | Description |
| --- | --- | --- | --- | --- |
| mode | Wait Mode | select | yes | - |
| waitFor | Wait for... | string | no | Component will wait for a fixed amount of time before emitting the event forward.

Supports expressions and expects integer.

Example expressions:
&#123;&#123;$.wait_time&#125;&#125;
&#123;&#123;$.wait_time + 5&#125;&#125;
&#123;&#123;$.status == "urgent" ? 0 : 30&#125;&#125; |
| unit | Unit | select | no | Time unit for the interval |
| waitUntil | Wait until | string | no | Component will countdown until the provided date/time before emitting an event forward.

Supports expressions and expects date in [ISO 8601](https://www.timestamp-converter.com/) format.

Example expressions:
&#123;&#123;$.run_time&#125;&#125;
&#123;&#123;$.run_time.In(timezone("UTC"))&#125;&#125;
&#123;&#123;$.run_time + duration("48h")&#125;&#125; |

### Example Output

```json
{
  "data": {
    "actor": {
      "display_name": "Alex Doe",
      "email": "alex@example.com"
    },
    "finished_at": "2024-01-01T12:05:00Z",
    "reason": "timeout",
    "result": "completed",
    "started_at": "2024-01-01T12:00:00Z"
  },
  "timestamp": "2026-01-16T17:56:16.680755501Z",
  "type": "wait.finished"
}
```

<a id="schedule"></a>

## Schedule

Start a new execution chain on a schedule

### Configuration

| Name | Label | Type | Required | Description |
| --- | --- | --- | --- | --- |
| type | Frequency | select | yes | - |
| timezone | Timezone | timezone | no | Timezone offset for scheduling calculations (default: your current timezone) |
| minutesInterval | Minutes between triggers | number | no | Number of minutes between triggers (1-59) |
| hoursInterval | Hours between triggers | number | no | Number of hours between triggers (1-23) |
| daysInterval | Days between triggers | number | no | Number of days between triggers (1-31) |
| weeksInterval | Weeks between triggers | number | no | Number of weeks between triggers (1-52) |
| monthsInterval | Months between triggers | number | no | Number of months between triggers (1-24) |
| dayOfMonth | Trigger on day of the month | number | no | Day of the month (1-31) |
| weekDays | Trigger on days of the week | multi-select | no | Select which days of the week to trigger |
| hour | Trigger at hour | number | no | Hour of the day (0-23) |
| minute | Trigger at minute | number | no | Minute of the hour (0-59) |
| cronExpression | Cron Expression | cron | no | Cron expression in 5-field (e.g., '30 14 * * MON-FRI') or 6-field (e.g., '0 30 14 * * MON-FRI') format. Valid wildcards: * , - / |
| customName | Run title (optional) | string | no | Optional run title template. Supports expressions like &#123;&#123; $.data &#125;&#125;. |

### Example Data

```json
{
  "calendar": {
    "day": "1",
    "hour": "09",
    "minute": "00",
    "month": "January",
    "second": "00",
    "week_day": "Monday",
    "year": "2024"
  },
  "timezone": "+00:00"
}
```
