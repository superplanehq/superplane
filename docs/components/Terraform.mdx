## On Run Needs Attention

Trigger a workflow when a Terraform Run is paused for policy overrides or approval.

## On Run Event

Trigger a workflow when a Terraform Run transitions to selected states.

<a id="apply-run"></a>

## Apply Run

Applies a run that is paused in 'needs attention' or 'planned'.

### Example Output

```json
{
  "action": "applied",
  "runId": "run-xxxxxx"
}
```

<a id="discard-run"></a>

## Discard Run

Discards a pending or planned run.

### Example Output

```json
{
  "action": "discarded",
  "runId": "run-xxxxxx"
}
```

<a id="override-policy"></a>

## Override Policy

Overrides a failed Sentinel policy block.

### Example Output

```json

  "action": "overridden",

}
```

## Queue Run


<a id="read-run-details"></a>

## Read Run Details

Retrieves comprehensive details and status about a run.

### Example Output

```json
{
  "createdAt": "2024-01-01T12:00:00Z",
  "message": "Queued manually",
  "runId": "run-xxxxxx",
  "status": "planned"
}
```

<a id="track-run"></a>

## Track Run

The Track Run component monitors a Terraform run and tracks its progression through states.

### States Tracked

- **Pending** → **Planning** → **Planned** → **Applying** → **Applied**
- Or: **Planned** → **Discarded** / **Errored** / **Canceled**

### Output Channels

- **Completed**: Run finished successfully (applied or planned_and_finished)
- **Failed**: Run errored, was canceled, or was discarded
- **Needs Attention**: Run requires manual confirmation or policy override

### Use Cases

- Track infrastructure changes through their lifecycle
- Get notified when runs complete or fail
- Chain actions based on run completion

### Example Output

```json
{
  "finalStatus": "applied",
  "runId": "run-xxxxx",
  "stateHistory": [
    {
      "status": "pending",
      "timestamp": "2024-01-01T12:00:00Z"
    },
    {
      "status": "planning",
      "timestamp": "2024-01-01T12:00:05Z"
    },
    {
      "status": "planned",
      "timestamp": "2024-01-01T12:01:00Z"
    },
    {
      "status": "applying",
      "timestamp": "2024-01-01T12:01:05Z"
    },
    {
      "status": "applied",
      "timestamp": "2024-01-01T12:02:00Z"
    }
  ]
}
```
