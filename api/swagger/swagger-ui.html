SuperPlane Public API

Automate canvases, executions, and organization operations without the UI.

The SuperPlane API lets teams provision canvases, update configuration as code, trigger runs, inspect execution state, manage service accounts, and automate organization settings from scripts, CI/CD jobs, and internal tooling.

  • Canvas lifecycle: create, update, delete, version, duplicate, and inspect canvases programmatically.
  • Execution control: list events, inspect outputs, invoke actions, resolve failures, and cancel runs.
  • Organization automation: manage integrations, invite flows, service accounts, users, roles, and groups.
  • Configuration-as-code ready: pair the API with GitOps workflows and the SuperPlane CLI.

Authentication

SuperPlane uses bearer token authentication for API requests.

Header
Authorization: Bearer <token>
Token sources
Personal API tokens from Settings > Profile or org-scoped tokens from Settings > Service accounts.
CLI behavior
The SuperPlane CLI uses the same bearer header, so requests in these docs match what the CLI sends.

Quickstart

Use these steps to make authenticated requests directly from the browser docs.

  1. Create a token from Settings > Profile or Settings > Service accounts.
  2. Paste the token into the bearer token field on this page and apply it.
  3. Open any operation below, click Try it out, and send the request.
curl -X GET \ "$API_BASE/api/v1/me" \ -H "Authorization: Bearer $SUPERPLANE_API_TOKEN" \ -H "Accept: application/json"

Use a Personal Token

Personal tokens are useful for local scripts, prototyping, and developer workflows tied to a human user.

  1. In the SuperPlane UI, go to Settings > Profile.
  2. Select Generate Token or Regenerate Token.
  3. Copy the token immediately. It is shown only once.

Regenerating a personal token invalidates the previous one.

Use a Service Account Token

Service accounts are the right choice for CI/CD, shared automation, and machine-to-machine access.

  1. In the SuperPlane UI, go to Settings > Service accounts.
  2. Create a service account and assign it a role.
  3. Generate or regenerate its API token and copy it immediately. It is shown only once.

Service account tokens are organization-scoped and should be preferred for production automation.

Starter Requests

These endpoints are a good entry point for building automation around canvases and organization management.

  • GET /api/v1/me to verify authentication and inspect the current user.
  • GET /api/v1/canvases to list canvases available to the token.
  • POST /api/v1/canvases to provision a new canvas.
  • GET /api/v1/service-accounts to list automation identities in the organization.
  • GET /api/v1/organizations/{id}/integrations to inspect configured connections without exposing secrets.
curl -X GET \ "$API_BASE/api/v1/canvases" \ -H "Authorization: Bearer $SUPERPLANE_API_TOKEN" \ -H "Accept: application/json"
